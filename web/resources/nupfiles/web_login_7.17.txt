welcome 홈페이지 : 누구든 홈페이지에 방문하면 처음 보게 될 화면
web.xml(배치관리자)은 함부로 손대면 안되고 오타나면 홈페이지오류로 이어진다.

welcome파일은 무조건 content irectory파일에 들어있어야한다.

web 우클릭 > new JSP file > index.jsp

views 우클릭 > new folder > example 만들어서 어제 만들었던 웰컴파일 옮겼다

title 은 탭에 표시될 이름. 보통 사이트의 이름

context root : 웹어플리케이션의 루트까지 접근하기 위한 이름
	      이 이름을 실제 프로젝트명을 쓰면 해킹당한다.
	      그래서 다 가짜 이름을 사용한다. (우리는 test1이라고 설정함)
	      context는 웹어플리케이션을 의미 


http 프로토콜이 기본. 여기에 보안이 추가된 것이 https
공기업, 공공기관은 보안 프로토콜(https) 써야함
 > 우리는 기본인 http 사용하면 될 것 같다.

웹어플리케이션은 스스로 실행하지 못한다. 이것을 실행 시켜주는 프로그램은 왓스
왓스는 웹 어플리케이션 서버입니다. 
왓스가 하는 일이 뭔데요? 라고 구글링한 결과말고 스스로 이해한 내용으로 말하기.

톰캣에 등록된 웹어플리케이션을 실행시켜준다.
구동중인 서버의 아이피
각각 왓스들이 구동되고 있다.
포트번호는 톰캣이 정한다. 톰캣은 기본으로 8080으로 잡아준다. 한 컴퓨터 내에서 포트번호는 유일해야한다.
우리는 이미 8080쓰고 있어서 8888로 바꾸어줬다.
톰캣 3개 구동해도 상관없지만 포트번호만 다르면 된다.

아이피:포트번호:context root(test1) 로 주소 입력하면 들어갈 수 있다.
포트번호로 찾아간다? 보통 80번으로 잡는다.

도메인 = 아이피:포트번호:context root(test1)  url 정보를 가리는 이름.

바로 왓스에 올리는게 아니라 패키징을 한다. 
패키징은 일종의 압축으로 웹어플리케이션은 war 형태로 압축해서 
톰캣안의 root파일 안에 넣어준다.
	
규모에 따라 와르까지만 제공, 와르를 왓스에 넣기 까지 직원의 업무이다.
